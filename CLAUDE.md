# Instruções de fluxo de trabalho — UltraNote2

- **Não commitar automaticamente.** Só faça `git commit`/`git push` quando o usuário pedir explicitamente. Pode sugerir o momento certo (ex.: "quer que eu commite isso?"), mas não execute por conta própria.
- **Não copiar para o QNAP automaticamente.** Só rode o `robocopy` para `K:\ultranote-app` quando o usuário pedir. Pode sugerir.
- **Não testar no browser automaticamente.** Só rode verificação via preview/browser quando o usuário pedir. Por padrão, apenas atualize o código e suba/reinicie o servidor local para o usuário testar manualmente.
